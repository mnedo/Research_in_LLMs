# Репорт к Дз2 #
## Выбор env ##
В качестве игры-среда была выбрана задача из ЕГЭ по информатике под номером 4, поскольку в качестве одного из аспектов своей деятельности я преподаю достаточно большому количеству школьников. И по опыту - это одна из немногих задач в экзамене, с которыми LLM-модели справляются не всегда хорошо. Мне было интересно "дотюнить" модель и посмотреть, улучшит ли она свои результаты.

Условия игры: 

В замкнутой системе существует $N$ символов. Для некоторых из них ($\le N$) коды для кодирования известны, для остальных - нет. Необходимо определить минимально возможную суммарную длину для оставшихся кодов. Сообщение должно расшифровываться одназначно, поэтому коды по условиям задачи должны удовлетворять условию Фано. Условие Фано - никакой код не должен быть началом любого другого кода (так мы можем одназначно декодировать набор бит, начав сначала).

Решение:

Фактически, коды располагаются на двоичном дереве, где вершины родителя отличатюся концовкой - на 1/0. Если некоторую вершину занять под букву, то все потомки нельзя будет использовать в качестве кодов. Поскольку дерево бесконечное, решение заключается в эффективном выборе вершин на таком дереве.

Сложность и параметры задачи:
В задаче можно настроить: количество неизвестных кодов и количество кодов во всем сообщении. От этого зависит и сложность: 1 - 4-5 букв, лишь одна неизвестна: 10 - весь латинский алфавит, известна лишь пара-тройка кодов.

Например, Gpt-5.2-thinking тратит 1.5 минуты на решение задачи средней сложности:

<img src="https://github.com/mnedo/Research_in_LLMs/blob/4117ff0caaad254d0d5c2886e1a749948cdb04f4/hw2_RL_envs_test/images/Gpt52thinking.png" alt="Proof of thinking time" width="50%"/>


Для задач сложности 10, модель не всегда выдает по итогу правильный ответ. Даже код для решения этого номера она может показать не сразу верный.

## Промптинг ##
Модели было проблематично выдавать ответы в правильном формате. Часто она искала суммарную длину для всех кодов, а не только неизвестных. Поэтому была использована технология One-shot, с примером размышления - подсчета ответа - модель стала выдавать правильные ответы в два раза чаще.

## Награды ##
* Модель получала до +0.8 за использование всех структурных токенов (think, answer) из System prompt
* Модель получала +5 за правильные ответ на задачу
* Модель получала штраф в случае неправильного ответа в размере $\sqrt{|answer_{model} - answer_{true}|}$
