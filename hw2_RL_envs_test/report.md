# Репорт к Дз2 #
## Выбор env ##
В качестве игры-среда была выбрана задача из ЕГЭ по информатике под номером 4, поскольку в качестве одного из апсектов своей деятельности я преподаю достаточно большому количеству школников. И по опыту - это одна из немногих задач в экзамене, с которыми LLM-модели справляются не всегда хорошо. Мне было интересно "дотюнить" модель и посмотреть, улучшит ли она свои резльутаты.

Условия игры: 

В замкнутой системе существует $N$ символов. Для некоторых из них ($\le N$) коды для кодирования известны, для остальных - нет. Необходимо определить минимально возможную суммарную длину для оставшихся кодов. Сообщение должно расшифровываться одназанчно, поэтому коды по условиям задачи должны удоволетворять условию Фано. Условие Фано - никакой код не должен быть началом любого друго кода (так мы можем одназанчно декодировать набор бит, начав сначала).

Решение:

Фактически, коды располагаются на двоичном дереве, где вершины роидтеля отличаюся концовкой - на 1/0. Если некоторую вершину занять под букву, то все потомки нельзя будет использовать в качестве кодов. Поскольку дерево бесконечное, решение заключается в эффектином выборе вершин на таком дереве.
[]()

Сложность и параметры задачи:
В задаче можно настроить: количество неизвестных кодов и количество кодов во всем сообщение. От этого зависит и сложность: 1 - 4-5 букв, лишь одна неизвестна: 10 - весь латинский алфавит, известна лишь пара-тройка кодов.

Например, Gpt-5.2-thinking тратит 1.5 минуты на решение задачи средней сложности:
[]()

Для задач сложности 10, модель не всегда выдает по итогу правильный ответ. Даже код для решения этого номера она может показать не сразу верный.

## Промптинг ##
Модели было проблематично выдавать ответы в правильном формате. Часто она искала суммарную длину для всех кодов, а не только неизвестных. Поэтому была использована технология One-shot, с примером размышления - подсчета ответа - модель стала выдавать правильные ответы в два раза чаще.

## Награды ##
* Модель получала до +0.8 за использование всех структруных токенов (<think>, <answer>) из System prompt
* Модель получала +5 за правильные ответ на задачу
* Модель получала штраф в случае неправильного ответа в размере $\sqrt{|answer_{model} - answe_{true}|}$
  
## Результаты ##


Стоит отметить, что даже tuned-модели часто не удавалось довести размышления до конца. Если взять во внимание время размышлений передовых моделей над задачами даже сложности 1, стоило бы увеличить количество тоекнов, которое может генерировать модель, однако мы ограничены ресурсами Google Colab
